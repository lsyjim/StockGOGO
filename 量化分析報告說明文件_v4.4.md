**計算公式：**
```
總體評分 = 技術面分數 + 基本面分數 + 籌碼面分數
```

**各面向評分標準：**

| 面向 | 訊號 | 分數 |
|------|------|------|
| 技術面 | 偏多 | +30 |
| 技術面 | 中性 | +15 |
| 技術面 | 偏空 | +0 |
| 基本面 | 偏多 | +30 |
| 基本面 | 中性 | +15 |
| 基本面 | 偏空 | +0 |
| 籌碼面 | 籌碼集中/偏多 | +30 |
| 籌碼面 | 籌碼中性 | +20 |
| 籌碼面 | 籌碼分散/偏空 | +0 |

**評分解讀：**
- 90-100：極度看好
- 70-89：看好
- 50-69：中性
- 30-49：偏空
- 0-29：極度看空

## 5. 均值回歸與乖離分析

### 5.1 乖離率計算

```
20MA乖離 = (現價 - MA20) / MA20 × 100%
60MA乖離 = (現價 - MA60) / MA60 × 100%
```

### 5.2 左側買進訊號（漲多預判拉回）

| 條件 | 閾值 |
|------|------|
| 負乖離過大 | 乖離 < -10% |
| RSI 超賣 | RSI < 25 |
| 止跌 K 線 | 長下影線（下影線 > 實體×2） |
| 止跌爆量 | 成交量 > 5日均量×1.5 |

觸發任一條件即產生左側買進訊號。

### 5.3 左側賣出訊號（跌多預判反彈）

| 條件 | 閾值 |
|------|------|
| 正乖離過大 | 乖離 > 15% |
| RSI 超買 | RSI > 75 |
| 頭部 K 線 | 長上影線（上影線 > 實體×2） |
| 高檔爆量不漲 | 成交量 > 5日均量×2 但漲幅 < 1% |

觸發任一條件即產生左側賣出訊號。

### 5.4 雙軌出場策略

| 策略 | 說明 | 適用對象 |
|------|------|----------|
| 防守型出場 | 收盤跌破 MA20 | 波段持有者 |
| 積極型停利 | 正乖離 > 15% 或 盈虧比 > 3 | 短線/資金效率派 |

| VP12 | 跳空下跌帶量 | 向下跳空 且 成交量 > 均量×1.5 | ★★★★★ |

### 6.3 量價分數計算

```
量價分數 = Σ(多方情境分數) - Σ(空方情境分數)

每個情境分數 = 嚴重度 × 方向權重
- 嚴重度：1-5（對應星數）
- 方向權重：多方 +1，空方 -1
```

**量價分數解讀：**
- > +20：強勁多方量價
- +10 ~ +20：偏多
- -10 ~ +10：中性
- -20 ~ -10：偏空
- < -20：強勁空方量價

## 7. 風險管理分析

### 7.1 ATR 停損停利計算

```
ATR = 14日平均真實波幅
建議停損 = 進場價 - ATR × 2.0
建議停利 = 進場價 + ATR × 2.0 × R倍數目標
```

**預設參數：**
- ATR 週期：14 日
- 停損倍數：2.0 倍 ATR
- R 倍數目標：2.0（即停利空間 = 停損空間 × 2）

### 7.2 風險回報比（RR Ratio）

```
風險回報比 = (停利價 - 現價) / (現價 - 停損價)
```

**判斷標準：**
- RR < 1.5：盈虧比不佳，降級為觀望
- RR 1.5-2.0：可接受
- RR > 2.0：良好
- RR > 3.0：極佳（黃金買點條件之一）

### 7.3 凱利倉位計算

```
建議倉位 = 資金 × 風險比例 / (ATR × 2.0)
預設風險比例 = 1%（每筆交易最大損失）
```

### 7.4 流動性檢查

| 指標 | 最低要求 | 說明 |
|------|----------|------|
| 20日均量 | > 500 張 | 確保可順利進出 |
| 20日均成交金額 | > 500萬 | 避免流動性風險 |

- < 12：偏低，偏多訊號

- > 25：偏高，偏空訊號

**PE 歷史百分位判斷：**
- < 20%：歷史低檔，偏多
- 20%-80%：正常
- > 80%：歷史高檔，偏空

### 8.3 籌碼面分析

| 指標 | 說明 |

| 投信買賣超 | 投信今日淨買賣張數 |
| 自營商買賣超 | 自營商今日淨買賣張數 |

**籌碼面綜合訊號：**

- 籌碼偏多：外資或投信連續買超
- 籌碼中性：買賣互見
- 籌碼偏空：外資或投信連續賣超
- 籌碼分散：外資+投信同步連續賣超 ≥2天

## 9. 信心度說明

### 9.1 信心度定義

信心度反映系統對當前建議的確定程度：

| 信心度 | 含義 | 觸發條件 |
|--------|------|----------|
| High | 高確定性 | 訊號明確、無矛盾、濾網未降級 |
| Medium | 中等確定性 | 訊號正常但有些不確定因素 |
| Low | 低確定性 | 被濾網降級、訊號矛盾、條件邊界 |

### 9.2 信心度調整規則

**升級為 High 的情況：**
1. 場景 B（黃金買點）且 RR > 1.5
2. 場景 D（空頭確認）趨勢明確
3. 量價分析確認「帶量突破」
4. 量價分析確認「放量跌破」

**降級為 Low 的情況：**
1. 風險回報比 < 1.5
2. 創高量縮（疑似假突破）
3. 放量不漲（派發跡象）
4. 場景 X（無法判斷）
5. 濾網觸發降級

### 9.3 如何使用信心度

| 信心度 | 建議操作 |
|--------|----------|
| High | 可依建議執行，倉位可較大 |
| Medium | 可參考執行，建議分批或觀望 |
| Low | 謹慎對待，建議等待更明確訊號 |

## 10. PE 為負值的原因與處理

### 10.1 問題說明

在 6770 力積電的報告中顯示「預估PE：-138.53」，這是因為：

**Forward PE 計算公式：**
```
Forward PE = 當前股價 / 預估未來12個月 EPS
```

### 10.2 負值原因

當 **預估 EPS 為負數**（即公司預估未來會虧損）時：
```
Forward PE = 56.4 / (-0.407) ≈ -138.5
```

這表示：
1. **公司目前虧損或預估未來虧損**
2. **Yahoo Finance 提供的預估 EPS 為負值**
3. **這是正常的計算結果，不是程式錯誤**

### 10.3 如何解讀負 PE

| PE 值 | 解讀 |
|-------|------|
| 正值且 < 15 | 估值偏低 |
| 正值且 15-25 | 估值正常 |
| 正值且 > 25 | 估值偏高 |
| 負值 | 公司虧損，PE 無意義 |
| N/A | 無資料 |

### 10.4 PE 負值時的系統處理（v4.4.2 修正）

當 PE 為負值時，系統會：

1. **基本面分析**：
   - 跳過 PE 判斷
   - 改用 PB（股價淨值比）作為替代指標
   - 顯示「公司虧損」提示

2. **長線建議**：根據 PB 和其他指標判斷

| PB 值 | 技術/籌碼面 | 長線建議 |
|-------|-------------|----------|
| < 1.0 | - | 長線觀察（股價低於淨值，可關注轉機）|
| 1.0-2.0 | - | 長線謹慎（需觀察獲利改善）|
| > 2.0 | - | 長線避開（PB偏高，風險大）|
| N/A | 偏多 | 長線觀察（可能有轉機題材）|
| N/A | 偏空 | 長線避開（風險較高）|
| N/A | 中性 | 長線謹慎（需觀察獲利改善）|

3. **報告顯示**：
   - PE 後方顯示 ⚠️ 警告標記
   - 評估理由顯示「公司虧損」

### 10.5 虧損公司的估值建議

對於虧損公司，應更重視：
- 營收成長趨勢
- 現金流狀況
- 技術面支撐壓力
- 籌碼面動向
- PB（股價淨值比）

## 附錄 B：v4.4.3 新增功能說明

### B.1 動態成交量閾值（Z-Score）

傳統固定倍數閾值（如 2.0 倍均量）對於大型權值股難以觸發。v4.4.3 改用 Z-Score：

```
成交量 Z-Score = (當日成交量 - 60日均量) / 60日標準差

判斷標準：
- Z-Score > 2.5：爆量 (Volume Spike)
- Z-Score > 3.0：大爆量 (Heavy Volume)
```

### B.2 布林通道帶寬壓縮警示

當布林帶寬處於歷史低檔（< 20% 百分位）時，表示波動率極度壓縮，即將出現突破：

```
帶寬 = (上軌 - 下軌) / 中軌 × 100%

若帶寬百分位 < 20%：
⚠️ 波動率壓縮極致，即將變盤 (Volatility Squeeze)
```

### B.3 時間停損機制

除了價格停損外，v4.4.3 新增時間維度的停損：


### B.6 v4.4.4 否決權機制（Veto Power）

v4.4.4 引入否決權機制，解決「訊號觸發但條件不佳」的矛盾問題：

**問題場景**：
```
報告上段：場景 A - 多頭過熱，建議持股觀望
報告下段：今日建議買進 ← 矛盾！
```

**修正後邏輯**：
```
┌─────────────────────────────────────────────────────────┐
│                   否決權優先級順序                        │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  Level 1: 賣出訊號（最高優先）                           │
│           → 直接輸出賣出建議                             │
│                                                         │
│  Level 2: 阻擋條件（否決權）                             │
│           ├── 場景 A 過熱 → ⚠️ 禁止追價                 │
│           ├── RR < 1.5   → ⏸️ 風險過高                  │
│           ├── PE<0 且高乖離 → ⚠️ 投機股風險             │
│           └── RSI > 80   → ⏸️ 短線回檔風險              │
│           （即使買訊觸發，也會被否決）                    │
│                                                         │
│  Level 3: 買進訊號（無阻擋時才執行）                     │
│           → 檢查條件後輸出買進建議                       │
│                                                         │
│  Level 4: 無訊號（最低優先）                             │
│           → 輸出觀望建議                                 │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

**報告新增欄位**：
```
✅ 執行條件檢核：
   ✓ 訊號觸發：是
   ✓ 盈虧比 ≥ 1.5：2.00
   ✓ 非過熱場景：是       ← v4.4.4 新增
   ✓ 信心度：High
   ✓ 流動性：通過
```

**修正後範例**：
```
【場景 A 過熱情況】
📡 昨日訊號：🟢 買進訊號觸發：技術面+籌碼面雙多
📌 今日建議：⚠️ 短線嚴重過熱，禁止追價，建議持股續抱或停利
🔔 結論：⚠️ 過熱警示：短線追價風險極高，建議持股觀望或停利
```
